# Lamda表达式

用于简化代码

()->{}

为什么要用？

1. 避免匿名内部类定义过多
2. 可以让你的代码看起来简洁
3. 去掉了一堆没有意义的代码，只留下核心的逻辑

理解函数式接口是学习lambda表达式的关键

匿名内部类是new一个接口或父类，没有类的名称，只能借助接口或者父类

##  函数式接口的定义

1. 任何接口，只包含唯一一个抽象方法，就是一个函数式接口
2. 对于函数式接口，我们可以通过lambda表达式来创建该接口的对象。



```java
Like like;
//匿名内部类
like = new ILike(){
    public void lambda(){
        //
    }
}
like.lambda();

//用lambda简化
like = ()->{
    //
};
like.lambda();
```



如果只有单个参数，括号也可以简化 a->{}

如果只有一行代码，中括号也可以简化

多个参数也可以去掉参数类型
